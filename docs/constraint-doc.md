## 强制约束条件（AI 必须遵守）

### 1. 路径穿越修复约束

- **必须** 使用 `os.path.realpath()` 规范化接收到的文件路径
- **必须** 验证最终路径在允许的输出目录（`OUTPUT_DIR`）范围内
- **禁止** 仅靠检查 `..` 字符串来防护路径穿越
- **禁止** 使用用户输入的原始字符串直接构造文件路径

### 2. 文件写入安全约束

- **必须** 使用 `with open()` 语句（上下文管理器）处理文件，确保文件句柄安全关闭
- **必须** 添加文件大小限制（建议 `MAX_FILE_SIZE = 100 * 1024 * 1024`，即 100MB）
- **必须** 对接收文件的扩展名做白名单校验（允许 `.txt`、`.pdf`、`.py`、`.json`、`.md`、`.csv`）
- **禁止** 接收并写入无扩展名或不在白名单内的文件

### 3. 异常处理约束

- **必须** 捕获具体异常类型（`socket.timeout`、`OSError`、`ConnectionRefusedError` 等）
- **禁止** 使用裸 `except Exception` 吞掉所有异常而不重新抛出
- **必须** 为安全相关异常（路径校验失败、文件超限）添加明确的日志记录

### 4. 代码风格约束

- **必须** 保持原有功能逻辑不变（只加安全层，不重写业务逻辑）
- **必须** 用语义化常量替代魔法数字
- **禁止** 引入新的第三方依赖库（只用 Python 标准库）
- **禁止** 使用 `eval()`、`exec()` 或 `subprocess` 处理任何输入数据

### 5. 文档约束

- **必须** 在修复的函数/逻辑处添加说明注释，标注修复了哪个安全风险（引用 R-01~R-04 编号）

# 项目约束文档（多模态安全测试框架 - AMIA防御）

## 项目背景

这是《信息安全管理》课程的实践作业，仓库为 `renyanbinbin/CyberSec-Practice-2026`。  
我们正在对 `成员代码/renyanbin/amia_defense_test.py`（一个基于AMIA防御算法的多模态大模型安全自动化测试框架）进行安全整改。

## 强制约束条件（AI 必须遵守）

### 1. 文件读取与路径穿越修复约束（对应 R-01：图片加载路径遍历）

- **必须** 使用 `os.path.realpath()` 规范化从 `data/figstep_samples.jsonl` 中读取的 `img_path` 字段。
- **必须** 验证最终绝对路径位于指定的安全数据根目录（如 `os.path.join(os.getcwd(), "data")`）范围内。
- **禁止** 仅靠检查 `..` 字符串来防护路径穿越（例如 `if ".." in img_path` 是不充分的）。
- **禁止** 使用从外部数据文件读取的原始字符串直接构造 `Image.open()` 参数而不做规范化校验。
- **必须** 对 `DATA_INPUT` 主数据文件路径同样做路径规范化校验。

### 2. 模块加载安全与导入路径修复约束（对应 R-02：`sys.path.append` 污染）

- **必须** 移除 `sys.path.append("../../")` 这一行硬编码的相对路径导入方式。
- **必须** 改用基于项目的绝对导入（例如 `from src.amia import AMIA`）或将模块安装为包（`pip install -e .`）。
- **禁止** 在运行时动态添加用户可控或相对层级过深的路径到 `sys.path`。
- **若无法修改项目结构**，必须将 `sys.path.append` 中的相对路径转换为基于当前脚本所在目录的绝对路径，并严格校验该目录是否属于项目预期范围。

### 3. 输入数据安全与资源耗尽防护约束（对应 R-03：Prompt注入，R-04：资源耗尽）

- **必须** 在调用 `Image.open()` 后，校验图片像素尺寸，**必须** 设置 `MAX_IMAGE_PIXELS = 4096 * 4096`（或等效 16MP），拒绝超大图片。
- **必须** 限制输入提示词（`s["text"]`）的长度，**必须** 设置 `MAX_PROMPT_LENGTH = 4096` 字符，超过则截断或跳过并记录日志。
- **必须** 对接收到的图片文件大小做校验（`os.path.getsize`），**必须** 设置 `MAX_IMAGE_FILE_SIZE = 50 * 1024 * 1024`（50MB）。
- **禁止** 将未经验证的 JSON 字段直接传入模型推理接口而不做边界检查。
- **（可选推荐）** 为 `vlm.generate()` 调用添加 `timeout` 超时机制（如使用 `threading` 或 `signal`），防止模型推理无限阻塞。

### 4. 异常处理与日志安全约束（对应 R-05：异常捕获不完整，R-06：日志泄露，R-08：调试代码残留）

- **必须** 为模型推理（`vlm.generate` 或 `defender.defend`）和文件写入添加细粒度的 `try-except`，**必须** 捕获具体异常类型（如 `torch.cuda.OutOfMemoryError`、`PIL.UnidentifiedImageError`、`IOError`、`json.JSONDecodeError`）。
- **禁止** 使用裸 `except Exception` 继续执行而不记录错误上下文（除非在最外层兜底并明确记录）。
- **必须** 删除所有调试用的硬编码开关（如注释掉的 `debug_idx` 和 `# if i != debug_idx: continue`）。
- **必须** 对写入结果文件（`RESULT_OUT`）和打印到控制台的日志进行**脱敏处理**：输出（`ans`）和意图分析（`reason`）若超过 200 字符，仅记录前 200 字符加 `...`；**禁止** 在正式运行日志中打印完整的 `reason` 详细内容（该内容可能包含模型内部推理痕迹）。
- **必须** 在路径校验失败、文件超限、图片格式异常时，使用 `logging` 模块（配置 `WARNING` 级别）记录安全事件，而不仅仅是 `print`。

### 5. 文件写入与输出安全约束（对应 R-07：符号链接攻击）

- **必须** 在写入 `results/defense_results_v3.jsonl` 前，校验 `results` 目录是否为符号链接（使用 `os.path.islink()` 或 `os.path.realpath` 比较），防止覆盖系统敏感文件。
- **必须** 设置单条日志记录的大小上限（如 `MAX_LOG_ENTRY_SIZE = 10 * 1024`），防止恶意样本导致结果文件无限膨胀。
- **必须** 使用 `with open()` 上下文管理器写入文件，确保文件句柄及时关闭。

### 6. 代码风格与常量语义化约束

- **必须** 用语义化常量替换所有魔法数字（如图片像素限制、提示长度限制、文件大小限制）。
- **禁止** 引入新的第三方依赖库（仅限使用 Python 标准库 + 已存在的 `torch`、`PIL`、`transformers` 等）。
- **禁止** 使用 `eval()`、`exec()` 或 `subprocess` 处理任何来自数据文件的输入。

### 7. 文档与注释约束

- **必须** 在每一处修复的逻辑旁添加说明注释，标注修复了哪个安全风险（引用 R-01~R-08 编号），例如：`# FIXED R-01: 路径遍历防护`。
- **必须** 在文件头部更新修改记录，注明整改日期和修复的风险项。
